# Authority Map: Atlas Diligence

| Source ID | Authority | Basis | Claim Types It Can Support |
|---|---|---|---|
| S01 | supporting | CIM is prepared for diligence context but is not the source of record for customer metrics. | Company narrative, management-positioned market claims |
| S02 | supporting | Call notes capture management commentary but are not independently approved. | Management commentary, qualitative pipeline context |
| S03 | authoritative | Customer export appears to come from CRM export, pending RevOps confirmation. | Customer count, ARR, churn fields |
| S04 | superseded | File name and date suggest this is an older model. | Historical context only until current model is confirmed |
| S05 | background | Market summary lacks underlying source. | Market context only |

## Stale Or Superseded Sources

- `S04` should not support current figures unless finance confirms it is still valid.

## Source Of Record Questions

- RevOps must confirm whether `S03` is the source of record for customer metrics.
- Finance must identify the current approved model for any model-derived outputs.
