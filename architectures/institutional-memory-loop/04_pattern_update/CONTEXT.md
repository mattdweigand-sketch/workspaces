# Stage 04: Pattern Update

## Purpose

Write the validated lesson and update reusable patterns.

## Inputs

- Event record.
- Interpretation.
- Validation note.
- `_config/memory-schema.md`
- `_store/patterns.md`

## Process

1. Write a structured memory record.
2. Add tags from the taxonomy.
3. Update `patterns.md` only when the validated lesson changes the pattern.
4. Note where future workflows should read this memory.
5. Keep unvalidated claims out of the store.

## Output

Write record to `_store/records/[name]-[date].md` and update `_store/patterns.md`.

## Done Looks Like

The next similar case can use the lesson without re-running the post-mortem.
