# Stage 02: Interpretation

## Purpose

Propose what the event means while keeping facts, stated reasons, and assessed causes separate.

## Inputs

- `01_event_capture/output/event-record-[name]-[date].md`
- `_config/taxonomy.md`
- `_store/patterns.md`

## Process

1. Summarize facts.
2. Capture stated reasons.
3. Propose assessed causes with evidence.
4. Compare to existing patterns.
5. List what remains uncertain.

## Output

Write to `02_interpretation/output/interpretation-[name]-[date].md`.

## Done Looks Like

The proposed lesson is explicit, evidence-based, and not yet treated as validated truth.
