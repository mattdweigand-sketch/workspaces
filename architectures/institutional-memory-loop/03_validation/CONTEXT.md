# Stage 03: Validation

## Purpose

Require human validation before causal claims enter memory.

## Inputs

- Event record.
- Interpretation.
- `_config/validation-rules.md`

## Process

1. Present proposed causal claims.
2. Ask the named owner to approve, revise, or reject each claim.
3. Record confidence and dissent.
4. Mark any lesson as provisional if needed.
5. Do not update patterns until validation is complete.

## Output

Write to `03_validation/output/validation-note-[name]-[date].md`.

## Done Looks Like

The memory store only receives claims a human has approved or marked provisional.
