# Thresholds — Senior Backend Engineer Hiring Bar

The testable go/no-go rules the evaluate and decide stages apply. L3 reference, stable across
candidates. The point is consistency: the same candidate profile gets the same answer no matter
who runs the loop.

## Automatic Disqualifiers

Any single one ends the evaluation as a decline.

- **Any "Weak" on a core dimension** (Technical depth or System design). The role cannot succeed
  without these; a strong showing elsewhere does not buy them back.
- **Two or more "Weak" ratings** across the four dimensions, regardless of which.
- **A confirmed integrity flag** — fabricated experience, plagiarized take-home, or a
  reference-check contradiction on a material claim.
- **A hard requirement miss** — work authorization the role cannot sponsor, or an unbridgeable
  location/timezone conflict with the team's core hours.

State each as a yes/no test; a disqualifier is automatic and is not overridden by a strong
interview elsewhere.

## Weighing Criteria (when no disqualifier trips)

- **Strong on both core dimensions + no worse than Mixed elsewhere** → advance to offer.
- **Strong on one core dimension, Mixed on the other** → advance only with a named rationale
  from the hiring manager (e.g., the Mixed dimension is coachable and a strong mentor is on the
  team).
- **Mixed on both core dimensions** → decline, or need-more if a specific signal was missed.
- **Strong across the board** → advance; consider whether the level is actually right (a staff
  signal routes to the staff scorecard, not an over-leveled offer here).

## Need-More Triggers

Reserved for a specific, identifiable gap whose answer would flip the call — not general
uncertainty. Examples: the system-design interview ran short and the signal is genuinely
incomplete; one debrief is missing; a reference is pending on a claim that is decisive either
way. "I'm not sure" is not need-more; a named missing fact is.

## Decline-Reason Taxonomy

Tag every decline with one, so the pipeline's pattern is visible over time:
`core-depth`, `core-design`, `level-mismatch`, `collaboration`, `delivery`, `integrity`,
`requirement`, `withdrew`.
