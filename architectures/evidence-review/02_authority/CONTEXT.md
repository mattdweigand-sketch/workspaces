# Stage 02: Authority

## Purpose

Rank each source by authority before relying on it.

## Inputs

- `01_inventory/output/source-inventory-[name]-[date].md`
- `_config/authority-ladder.md`

## Process

1. Classify each source as authoritative, supporting, background, or superseded.
2. Explain why each source was classified that way.
3. Flag version families and likely stale files.
4. Identify which source should be cited for each major claim type.
5. Do not reconcile conflicting facts.

## Output

Write to `02_authority/output/authority-map-[name]-[date].md`.

## Done Looks Like

The team knows which sources can support claims and which cannot.
