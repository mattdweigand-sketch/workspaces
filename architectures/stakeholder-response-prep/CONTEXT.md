# Architecture: Stakeholder Response Prep

## Overview

Use this when the team needs to explain verified facts to a customer, executive, board, employee, partner, investor, or other stakeholder.

Do not use this to invent facts, bypass approval, send messages, or manage stakeholder records.

## Stage Map

| Stage | Purpose | Output |
|---|---|---|
| `01_verified_facts` | Gather approved facts and source notes. | Verified facts packet |
| `02_message_frame` | Frame the message and posture. | Message frame |
| `03_question_prep` | Prepare likely questions and responses. | Question prep |
| `04_review_packet` | Prepare the human-reviewed draft or brief. | Review packet |

## Optional Modules

Attach `modules/artifact-trust-layer/` when the response work produces LP narratives, investor updates, executive updates, board updates, customer-facing materials, decks, memos, or other stakeholder-facing artifacts.

When attached, add these optional outputs:

- Claim evidence map.
- Artifact review report.
- Human approval note.

## Platform Boundary

| Layer | Owner |
|---|---|
| Records, permissions, delivery, audit | Platform |
| Approved facts and figures | Platform / source owner |
| Narrative, question prep, tone, response posture | AI |
| Approval and send | Human |
