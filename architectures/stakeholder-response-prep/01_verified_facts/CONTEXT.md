# Stage 01: Verified Facts

## Purpose

Gather only the facts that have been verified by the platform or responsible owner.

## Inputs

- Platform export or approved source.
- Existing stakeholder context.
- `_config/claim-rules.md`
- `_config/platform-boundary.md`

## Process

1. List verified facts.
2. Tie each fact to its source.
3. Label preliminary, estimated, sensitive, or unapproved facts.
4. Exclude anything unsupported.
5. List open confirmations.

## Output

Write to `01_verified_facts/output/verified-facts-[name]-[date].md`.

## Done Looks Like

The message can be drafted without inventing or chasing facts.
