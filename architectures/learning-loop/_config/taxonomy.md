# Taxonomy

<!--
ANNOTATION: The fixed vocabulary for tagging each record, so the store can
aggregate. Patterns emerge by tag — "we lose limited processes to faster-moving
counterparties" is only visible if process type and decisive dimension are tagged
consistently. A free-text field will not aggregate; a controlled taxonomy will.

The counterparty/intermediary list is a controlled list on purpose: "whose
feedback proves reliable" and "which counterparties we win through" are two of
the highest-value patterns this store can surface, and both require a stable,
controlled vocabulary.

This is L3 reference, loaded in stages 01 and 03. Keep it stable; add categories

## Case Type
[A controlled list of case types the team competes for. Tag exactly one (or a
primary) per record. Examples: adapt to your domain.]

## Size Band
[Controlled bands so scale aggregates. Set the boundaries to your team.
Example bands:
- Small
- Medium
- Large
- Very large]

## Process Type
[How the case was run, as a controlled tag. Examples:
- Broadly marketed
- Limited / select
- Direct
- Bilateral / negotiated]

## Counterparty Type
[Who we competed against or worked with, as a controlled tag. Examples:
- Institutional
- Private / individual
- Incumbent
- New entrant]

## Counterparty / Intermediary
[A CONTROLLED list of the counterparties and intermediaries you engage with
repeatedly. This is the highest-value tag for two patterns — which counterparties
you win through, and whose debrief feedback proves reliable — so keep it a fixed
list, not free text. Add a counterparty only when you have engaged with them.
"Direct / none" is a valid value.]

## Domain
[Controlled domain tags at the grain you actually compete in. Keep the list to the
areas you compete in; add a sub-domain only when you will query by it.]

## Outcome
[Won / Lost / Withdrew-late. Tag exactly one. Withdrew-late is a flagged sub-case
— a different lesson than a clean loss.]

## Decisive Dimension
[The single dimension that decided the outcome, as a controlled tag, so the store
can answer "what actually wins and loses our cases." Examples: price, certainty /
speed, structure, terms, reputation / relationship, timing. Tag the primary one;
note a secondary in the record if needed.]
