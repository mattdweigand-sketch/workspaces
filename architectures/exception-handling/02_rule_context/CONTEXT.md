# Stage 02: Rule Context

## Purpose

Identify the rules, policies, precedents, and constraints that apply.

## Inputs

- `01_exception_brief/output/exception-brief-[name]-[date].md`
- `_config/rule-sources.md`

## Process

1. List relevant rules and policy sources.
2. Note which parts clearly apply.
3. Note where the case is ambiguous.
4. Compare to precedent where available.
5. Identify mandatory escalations.

## Output

Write to `02_rule_context/output/rule-context-[name]-[date].md`.

## Done Looks Like

The exception is grounded in known rules before options are drafted.
