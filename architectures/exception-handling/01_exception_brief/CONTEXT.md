# Stage 01: Exception Brief

## Purpose

Explain what makes the case an exception.

## Inputs

- Case record or intake brief.
- Related platform status.
- `_config/exception-types.md`

## Process

1. State the normal path.
2. State why this case does not fit it.
3. List facts, assumptions, and missing context.
4. Identify affected stakeholders.
5. Mark urgency and risk.

## Output

Write to `01_exception_brief/output/exception-brief-[name]-[date].md`.

## Done Looks Like

The owner can understand the exception without reading the entire thread or record.
