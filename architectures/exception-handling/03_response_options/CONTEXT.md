# Stage 03: Response Options

## Purpose

Draft safe options for handling the exception.

## Inputs

- Exception brief.
- Rule context.
- `_config/escalation-rules.md`

## Process

1. Provide two or three viable paths.
2. State tradeoffs, risks, and required approvals for each.
3. Identify what cannot be done without platform or policy owner action.
4. Recommend a path when the evidence supports one.
5. Do not execute the action.

## Output

Write to `03_response_options/output/response-options-[name]-[date].md`.

## Done Looks Like

The human owner can choose a path without losing sight of constraints.
