# Stage 04: Escalation Handoff

## Purpose

Hand the exception to the right owner or platform workflow.

## Inputs

- Exception brief.
- Rule context.
- Response options.
- `_config/escalation-rules.md`
- `_config/platform-boundary.md`

## Process

1. Name the recommended owner.
2. Name the platform record or queue that should carry state.
3. Summarize the recommended path and open risks.
4. List approvals needed.
5. Produce the handoff note.

## Output

Write to `04_escalation_handoff/output/escalation-handoff-[name]-[date].md`.

## Done Looks Like

The exception is ready for human approval and platform-state update.
